대시보드 기능 한눈에 보기

1

Global Settings (전역 설정)

앱 전체에 적용되는 설정을 관리합니다.

  • Vercel 연동: 여기서 API Token을 한 번만 입력하면 모든 프로젝트의 배포 상태를 자동으로 감시합니다.
  • 매번 프로젝트마다 토큰을 넣을 필요가 없어 매우 편리합니다.
2

System Control (시스템 제어)

RAM Monitor: PC 전체 메모리가 90%를 넘으면 빨간색으로 경고합니다.

✨ Cleanup 기능 (강력 추천)
PC가 느려지거나 포트가 꼬였을 때 누르세요. 좀비(Zombie) Node 프로세스만 찾아내어 0.5초 만에 깔끔하게 정리해줍니다.
3

Resource Monitor (자원 감시)

각 프로젝트가 내 PC 메모리를 얼마나 먹고 있는지(MB) 실시간으로 보여줍니다.

실행 중인 Port와 윈도우 PID도 즉시 확인할 수 있습니다.

4

Action Controls (제어)

  • Start/Stop: 서버 실행 및 정지
  • 🚀 Browser: 로컬호스트 바로 열기
  • 💀 Kill Port: 포트 충돌(EADDRINUSE) 발생 시, 해당 포트를 점유한 프로세스를 강제 종료합니다.
5

Integrations (상태 표시)

  • Git: 현재 브랜치 및 변경 사항(↑Push / ↓Pull) 표시
  • 🌐 Vercel: 실시간 배포 상태 모니터링
    🔵: 빌드 중 / 🟢: 성공 / 🔴: 실패
6

Edit & Delete (관리)

연필 아이콘(✏️)을 눌러 프로젝트 경로, 실행 명령어, Vercel URL 등을 언제든 수정할 수 있습니다.

7

Smart Log Console (통합 로그)

모든 프로젝트의 로그가 이곳에 모입니다. 상단의 프로젝트 이름 탭을 클릭하여 보고 싶은 로그만 필터링하세요.

프로젝트 설정 상세 가이드

1

기본 정보

프로젝트 이름과 그룹, 실제 경로를 설정합니다. Node 버전 정보를 확인할 수 있습니다.

2

Cache Cleaning & Command

Auto-clean cache: 체크 시, 서버 시작 전에 .next, dist 등의 캐시 폴더를 자동으로 지웁니다. 꼬임 방지에 탁월합니다.

3

Vercel Auto-Detection

별도 설정 없이도 Deploy Config URL(4번)만 있으면 프로젝트 ID를 자동으로 찾아냅니다.

Global Token(전역 설정)을 사용 중이라면 이곳에 토큰을 입력할 필요가 없습니다. (Override 할 때만 입력)
4

Deployment URLs

GitHub 주소와 Vercel 배포 URL을 입력해두면 원클릭 이동 및 자동 상태 감시가 활성화됩니다.
